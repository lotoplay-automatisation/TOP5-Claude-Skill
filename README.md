# TOP5 Claude Skill — інструкція для Claude Code

> Цей README написаний **для тебе, Claude Code**, а не для людини. Якщо тебе попросили встановити або "підняти" цей скіл — виконай кроки нижче послідовно, кожен раз перевіряючи результат перед переходом далі. Не питай користувача про очевидні речі (шлях до `~/.claude/skills`, чи є Node) — перевір сам командами нижче. Питай користувача лише там, де явно позначено **[ПИТАННЯ КОРИСТУВАЧУ]**.

## Що це за скіл

`/top5` — автоматична генерація TOP5-шортсів (9:16 YouTube Shorts) під кожен GEO з однієї теми: локалізований сценарій → talking-avatar і фон/продуктові асети через Comfy Cloud → озвучка через ElevenLabs → фінальна збірка в HyperFrames (тайтл-плашка, персистентний купон, субтитри, safe zones). Повна логіка й правила — в `SKILL.md` у корені цього репо. **Прочитай `SKILL.md` повністю перш ніж щось встановлювати** — там контракт скіла (формат виклику, обов'язкові поля, QA-гейти).

## Крок 0 — визнач, куди ставити

- Якщо скіл потрібен користувачу глобально (в усіх проєктах) → ціль `~/.claude/skills/top5` (Windows: `%USERPROFILE%\.claude\skills\top5`).
- Якщо скіл потрібен лише в поточному проєкті → ціль `.claude/skills/top5` в корені поточного репозиторію.

Якщо незрозуміло з контексту розмови — **[ПИТАННЯ КОРИСТУВАЧУ]**: глобально чи лише для цього проєкту?

## Крок 1 — перевір передумови (виконай команди, не припускай)

Виконай і зафіксуй результат кожної команди перед тим як продовжити:

```bash
node -v                 # має бути >= 18. Якщо немає/старіша — попроси користувача поставити Node 18+
npm -v
claude --version        # переконайся що це саме Claude Code CLI/Desktop з підтримкою Skills
claude mcp list         # шукай сервер comfy-cloud (або схожу назву) в списку, статус має бути connected/ok
```

Якщо `comfy-cloud` MCP відсутній у списку — це блокер для генерації avatar/фону/продуктів. Повідом користувача, що MCP-сервер Comfy Cloud треба підключити окремо (`claude mcp add ...` або через Desktop → Settings → MCP Servers), і запитай, чи є в нього доступ/креденшели до Comfy Cloud, перш ніж продовжувати встановлення далі.

Перевір HyperFrames:

```bash
hyperframes --version
```

Якщо команда не знайдена встанови з цього репо:

https://github.com/heygen-com/hyperframes

## Крок 2 — встанови скіл у визначену з Кроку 0 теку

Перевір, чи ціль вже існує і чи це git-репозиторій (щоб не перезаписати чиюсь роботу):

```bash
# приклад для глобальної теки, підстав .claude/skills/top5 для проєктної
git status "$env:USERPROFILE\.claude\skills\top5" 2>$null
```

Якщо теки ще немає — клонуй:

```bash
git clone https://github.com/lotoplay-automatisation/TOP5-Claude-Skill.git "<ЦІЛЬОВА_ТЕКА_З_КРОКУ_0>"
```

Якщо git недоступний — завантаж репозиторій як ZIP і розпакуй у ту саму теку так, щоб `SKILL.md` лежав прямо в її корені (не в підпапці на рівень глибше).

Якщо тека вже існує і це git-репо цього ж скіла — онови замість повторного клонування:

```bash
cd "<ЦІЛЬОВА_ТЕКА>"
git pull
```

Якщо тека існує, але це **не** цей репозиторій — зупинись і **[ПИТАННЯ КОРИСТУВАЧУ]**: там вже щось лежить під назвою `top5`, перезаписати чи обрати іншу назву теки?

## Крок 3 — конфігурація

1. У теці скіла скопіюй приклад env-файлу, якщо `.env` ще немає:

   ```bash
   cp .env.example .env
   ```

2. Перевір вміст `.env.example` — на момент написання цього README там лише:

   ```
   ELEVENLABS_API_KEY=
   ```

   Якщо в `.env` це поле порожнє — **[ПИТАННЯ КОРИСТУВАЧУ]**: попроси ElevenLabs API key (https://elevenlabs.io/app/settings/api-keys) і впиши його в `.env`. Не вигадуй і не залишай приклад-значення.

3. Ще раз переконайся, що `comfy-cloud` MCP активний (`claude mcp list`) — без нього скіл не зможе генерувати avatar/фон/продукти навіть з правильним `.env`.

## Крок 4 — верифікація встановлення

Попроси користувача перезапустити Claude Code CLI/Desktop. Після того як користувач перезапустив Claude Code і написав щось в сесію перевір чи скіл реально встановлено:

   ```bash
   test -f "<ЦІЛЬОВА_ТЕКА>/SKILL.md" && echo OK
   ```

Якщо файл на місці — встановлення завершено. Тестовий прогін `/top5` робити не треба, це окрема дія користувача. Просто повідом, що скіл встановлено і готовий до використання командою `/top5`.

## Формат виклику (для довідки)

```
/top5 <тема> | <GEO1>,<GEO2>,...
```

Приклад:

```
/top5 розумний годинник | UA,PL
```

Результат — окрема тека проєкту на кожен GEO з усіма проміжними асетами та фінальним `.mp4`.
